✅ Component 1: Voice Capture
How It Works

The Voice Capture module listens through the user’s microphone and collects the raw audio input. In AURA, this component uses real-time noise reduction and silence detection so that only meaningful speech is captured. The system filters unwanted background sounds and prepares a clean audio signal before sending it to the Speech-to-Text engine.

Key Points

Listens to the user’s voice through the microphone in real time.

Removes background noise and unwanted disturbances.

Detects when the user starts and stops speaking.

Produces a clean audio signal for better recognition accuracy.

Works in both quiet and moderately noisy environments.

Acts as the first stage of AURA’s command processing.

✅ Component 2: Speech-to-Text (STT)
How It Works

Once voice is captured, AURA converts it into text. It uses Whisper (or online Google STT as fallback) to transcribe speech with high accuracy. The STT engine analyzes audio patterns, recognizes words, and produces text output, even if the user speaks quickly or with slight accents.

Key Points

Converts the captured voice into text using Whisper / Google STT.

Works even when pronunciation or phrasing varies slightly.

Whisper ensures offline speech recognition with high accuracy.

Google STT provides extremely accurate transcription online.

Handles pauses and natural speaking flow properly.

Provides the exact text that AURA uses for command detection.

✅ Component 3: Command Matching
How It Works

AURA compares the transcribed text with its internal list of commands. It uses fuzzy matching, keyword detection, and intent understanding to identify what the user wants to do. Even if the user phrases a command differently (“open Chrome” / “launch Chrome”), AURA can still recognize the intention.

Key Points

Detects the meaning of the user’s request using keywords.

Uses fuzzy logic to handle spelling or wording variations.

Matches user phrases with AURA’s internal command database.

Supports flexible phrasing (“open”, “launch”, “start”).

Extracts actionable intent from the recognized text.

Decides which backend function should be executed next.

✅ Component 4: Execute Function
How It Works

After matching a command, AURA triggers the correct backend function from its system modules. These functions perform real actions like opening apps, controlling system settings, searching files, running automation, or using AI learning modules. This is where the assistant interacts with the operating system.

Key Points

Executes the actual task based on matched command.

Connects with Windows APIs for system-level automation.

Runs modules like file search, OCR, macros, or shortcuts.

Ensures safe execution with error-handling in case of failures.

Supports multi-step operations (e.g., open app → maximize window).

All backend logic (main.py, aura_core.py) works in this part.

✅ Component 5: Voice Response
How It Works

Finally, AURA replies back to the user using Text-to-Speech (TTS). It takes the result of the executed function and converts it into spoken feedback. This helps users know whether a command was performed, what the result is, or if AURA needs more information.

Key Points

Converts text output into spoken voice using pyttsx3 or SAPI.

Gives confirmation of completed actions.

Provides results (e.g., “Your system brightness is now 70%”).

Asks follow-up questions when needed.

Maintains a natural flow of conversation.
